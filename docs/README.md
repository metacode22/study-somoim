## 빠른 참조

### 공통 용어 (registration 기반)

| 용어 | 영문 | 설명 |
|------|------|------|
| 소모임 | - | 취미/교류 중심 모임 |
| 스터디 | study | 학습/학업 중심 모임 |
| 챕터 | chapter | 활동 기간 단위 |
| 등록 | registration | 소모임/스터디를 생성하는 행위 |
| 등록 기간 | - | 소모임/스터디 등록이 가능한 기간 |
| 심사 대기 | **queued** | 등록 시 기본 상태 |
| 심사 승인 | **approved** | 참가 신청 가능 상태 |
| 심사 반려 | **returned** | 일반 사용자에게 미노출 |
| 신청 | application | 부원/순참 참가 신청 |
| 신청 기간 | - | 참가 신청이 가능한 기간 |
| 최종 등록 | - | 멤버 확정 후 활동 개시 선언 |

### 역할 코드 (registration 1.1)

| 코드 | 역할 | 설명 |
|------|------|------|
| `admin` | 관리자 | 심사(승인/반려), 전체 관리 |
| `leader` | 리더 | 등록·수정·멤버·활동 기록 관리 |
| `vice_leader` | 부리더 | 리더와 동일, 1개당 1명 |
| `member` | 부원 | 정규 멤버 (리더 선발) |
| `guest` | 순참 | 참관 형태 (사용자 신청) |

### 등록(registration) 입력 필드

- 리더 OT 참여 여부, **유형**, **카테고리**, 이름, 설명, 모임 주기, 모임 장소, **정원**
- 카테고리: 팀/파트/스쿼드, 운동, AI, 자기계발, 취미, 문화, 기타

### 신청(recruitment) 필터

- **신청 가능 여부**: 정규/순참/신규 입사자 / 자동 선발/선발 완료
- **요일**: 월·화·수·목·금·주말·비정기
- **카테고리**: (등록과 동일)

### 신청 플로우 요약

1. **신청 전** → 필터로 조회 → 상세 클릭 → 신청 확인 모달 → 정규/순참 버튼
2. **신청 후** → 요일별 신청 내역 → 신청 정보 모달(취소 가능, 선발 완료 전만)
3. **선발(리더)** → 신청자 리스트(이름, 팀, 선발/선발취소) → 선발 마감 → 최종 등록 모달

## 구현 시

- 기능·필드·플로우 추가 전 **해당 PRD 섹션**을 열어 확인
- PRD에 없는 요구사항은 구현하지 말고, **질문하거나 TODO**로 남기기
- 모호한 부분은 PRD 문구를 그대로 따르고, 필요 시 팀에 확인
