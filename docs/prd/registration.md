# 소모임 / 스터디 관리 시스템 기획 (MVP)

---

## 1. 사용자 역할 정의

### 1.1 역할 구조

| 상위 역할 | 하위 역할 | 코드 | 설명 |
| --- | --- | --- | --- |
| 관리자 | - | admin | 모든 기능 접근 및 수정 권한 보유<br/>Admin 권한 부여 및 관리자 페이지 접근 가능 |
| 사용자 | 리더 | leader | 소모임/스터디 등록 시 권한 부여<br/>등록한 소모임/스터디의 내용 및 멤버 수정 가능 |
|  | 부리더 | vice_leader | 리더가 지정 시 권한 부여<br/>리더와 동일한 권한 |
|  | 부원 | member | 사용자가 직접 선택하여 소모임/스터디에 신청 |
|  | 순참 | guest | 사용자가 직접 선택하여 소모임/스터디에 신청 |

---

### 1.2 사용자 공통 정의

- 대상: teamsparta Google 계정을 보유한 모든 사용자
- 사용자 정보
  - 이름 노출
  - 팀 정보 노출
- 사용자는 접속 후의 **행위에 따라 하위 역할이 분류**
- 한 사용자는 **복수의 하위 역할을 동시에 보유 가능**

---

### 1.3 하위 역할별 권한

#### 리더 (Leader)

- 소모임/스터디 개설 신청 시 자동 부여
- 권한
  - 소모임/스터디 내용 수정 및 삭제
  - 최종 등록 시 부리더 / 부원 / 순참 멤버 추가, 수정, 삭제
  - 해당 소모임/스터디의 활동 기록 작성, 수정, 삭제

#### 부리더 (Vice Leader)

- 리더가 지정 시 권한 부여
- 권한
  - 리더와 동일한 권한
- 제한
  - 소모임/스터디 1개당 1명만 지정 가능

#### 부원 (Member)

- 리더가 최종 등록 시 권한 부여
- 모집 기간 중 사용자가 선택하여 **임시 권한** 획득 가능

#### 순참 (Guest)

- 신청 기간 중 사용자가 선택하여 참가 신청
- 정식 멤버가 아닌 참관 형태

---

## 2. 유저 플로우

### 관리자

- 등록 기간
  - [심사 대기] 상태의 소모임/스터디 목록 확인
  - 심사 승인 / 심사 반려 처리
- 신청 기간
  - 운영 상태 모니터링

### 리더

- 등록 기간
  - 소모임/스터디 등록
  - 등록 내용 수정
- 신청 기간
  - 신청자 확인
  - 최종 멤버 구성 준비

### 부원 / 순참

- 등록 기간
  - 조회만 가능
- 신청 기간
  - 소모임/스터디 참가 신청

## 3. 용어 정의

| 용어 | 영문 | 설명 |
| --- | --- | --- |
| 소모임 | - | 취미/교류 중심 모임 |
| 스터디 | study | 학습/학업 중심 모임 |
| 챕터 | chapter | 활동 기간 단위 |
| 등록 | registration | 소모임/스터디를 생성하는 행위 |
| 등록 기간 | - | 소모임/스터디 등록이 가능한 기간 |
| 심사 대기 | queued | 등록 시 기본 상태<br/>관리자에게 검토 요청된 상태 |
| 심사 승인 | approved | 관리자 승인 완료 상태<br/>참가 신청 가능 |
| 심사 반려 | returned | 관리자 승인 거절 상태<br/>일반 사용자에게 노출되지 않음 |
| 신청 | application | 부원 또는 순참 자격으로 참가 신청 |
| 신청 기간 | - | 참가 신청이 가능한 기간 |
| 최종 등록 | - | 멤버 확정 후 활동 개시 선언 |

## 4. 기본 기능

- 관리자가 지정한 [등록 기간] 내에 사용자는 아래 정보를 입력하여 소모임/스터디를 등록할 수 있다
  - 리더 OT 참여 여부
  - 소모임/스터디 유형
  - 카테고리
  - 이름
  - 설명
  - 모임 주기
  - 모임 장소
  - 정원
- 사용자는 n개의 소모임/스터디를 등록할 수 있다
- 소모임/스터디 등록 시 상태는 기본적으로 [심사 대기]
- 관리자의 [심사 승인] 이후에만 부원/순참 모집 가능
- [심사 승인] 상태의 소모임/스터디만 신청 가능
- [심사 대기] 상태의 소모임/스터디 노출 여부는 정책 결정 필요

## 5. 기능 상세

### 5.1 사용자 권한

- 모든 사용자는 소모임/스터디를 등록할 수 있음
  - 인턴, 계약직 등의 필터링은 컬처팀에서 오프라인 관리
- 모든 사용자는 모든 소모임/스터디에 신청 가능

---

### 5.2 소모임 / 스터디 등록하기

#### 입력 필드

- 유형
- 카테고리
- 소모임/스터디 이름
- 소모임/스터디 내용
- 리더
- 모임 주기
- 모임 장소
- 정원
- 부원 / 순참 버튼 (신청 기간에만 활성화 → 신청 모달)
- 활동 기록 모음
- 목록으로 돌아가기 버튼

#### 기타

- 글자 수 제한
- 입력 타입 정의 (날짜, 요일, 드롭다운, 버튼 등)
- 유효성 검사
- 토스트 메시지
- 등록 후 상태 표시

---

### 5.3 소모임 / 스터디 수정하기

- [등록 기간] 내에만 전체 수정 가능
- [심사 승인] 이후에는 아래 항목만 수정 가능
  - 설명
  - 부리더

---

### 5.4 소모임 / 스터디 최종 등록

- 리더가 등록 기간 내에 수행
- 부리더 / 부원 / 순참 멤버 확정
- 활동 개시 선언
- 관리자에게 최종 멤버 정보 전달

---

### 5.5 소모임 / 스터디 심사 상태

- 심사 대기
- 심사 승인
- 심사 반려
  - 심사 반려된 소모임/스터디는 관리자 뷰에만 노출
  - 반려 후 수정 및 재심사 가능 여부 정책 결정 필요
